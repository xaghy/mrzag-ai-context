## 12. SESSION PROTOCOLS

> These protocols apply across all agents. Agents should follow them when triggered explicitly by Mohamed.
> Full execution details are in the spec: `docs/superpowers/specs/2026-03-18-agent-context-architecture-design.md`

### Trigger: "Sys Eng" (Session Start — Explicit Only)

When Mohamed says "sys eng" or "system engineer":
1. Check Windows Event Log — Errors/Warnings (Level 2, 3) from System and Application logs
2. Check RAM status (total / available)
3. Check storage health and free space
4. Read `SESSION_LOG.md` — all entries (max 3)
5. Output concise situation report — flag anomalies, list open issues

**For Claude Code:** Use PowerShell interop: `powershell.exe -Command "Get-WinEvent -LogName System -MaxEvents 20 | Where-Object {$_.Level -le 3}"`

### Trigger: "Session Complete" (Session End — Explicit Only)

When Mohamed says "session complete":
1. Prepend new entry to `SESSION_LOG.md` (delete oldest if > 3 entries)
2. Write compact JSON to Windows Event Log (source: `AgentSession`, EventId: 1001)
3. Prompt Mohamed to push to GitHub repo: `xaghy/mrzag-ai-context`

**SESSION_LOG.md entry format:**

```
---
## [YYYY-MM-DD] — [Agent Name] Session
**Open Issues:** [bullet list or "None"]
**Decisions Made:** [bullet list]
**Next Action:** [one line]
---
```

**Event Log write (PowerShell):**

```powershell
$msg = '{"date":"YYYY-MM-DD","agent":"[Agent]","open_issues":["..."],"decisions":["..."],"next_action":"..."}'
Write-EventLog -LogName Application -Source "AgentSession" -EventId 1001 -EntryType Information -Message $msg
```
